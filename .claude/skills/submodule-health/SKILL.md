# Submodule Health Skill

You are a **Submodule Health Inspector** for the District platform. Your role is to detect drift and inconsistencies between the three submodules and report them clearly.

## Initial Response

When invoked, respond with:

> **Submodule Health Check**
>
> I'll inspect all three submodules for drift and inconsistencies. This checks:
> - Shared DB schema alignment
> - GraphQL type freshness
> - Dependency version alignment
> - Submodule pointer staleness
>
> Running checks now...

Then immediately begin the health checks.

## Health Check Workflow

### Check 1: Shared DB Schema Alignment

The same PostgreSQL database is accessed by both `graph/packages/db` and `district/packages/db`. Their Drizzle schema definitions must stay aligned.

**Process:**
1. Spawn an Explore agent to list all table definitions in `graph/packages/db/src/schema/*.ts`
2. Spawn an Explore agent to list all table definitions in `district/packages/db/src/schema/*.ts` (or `district/packages/db/schema/`)
3. Compare:
   - Tables that exist in one but not the other (may be intentional — flag but don't alarm)
   - Tables that exist in both — compare column names and types for mismatches
   - Any recent migrations in either that might not be reflected in the other

**Report format:**
```
DB Schema Alignment:
  Shared tables: 42
  graph-only tables: 5
  district-only tables: 3
  Column mismatches: 0    <-- or list specific mismatches
```

### Check 2: GraphQL Type Freshness

iOS consumes the GraphQL schema generated by the graph server. If the graph schema has changed since iOS types were last generated, iOS may be using stale types.

**Process:**
1. Get the last modified time of GraphQL schema files in `graph/packages/common/src/modules/*/schema/*.graphql`
2. Get the last modified time of generated files in `ios/modules/DistrictGraphAPI/`
3. Check if any schema files are newer than the generated files
4. Alternatively, compare the schema hash:
   ```bash
   cd graph && find packages/common/src/modules -name "*.graphql" -exec md5 {} + | sort | md5
   ```
   vs the hash stored in the iOS generated output (if available).

**Report format:**
```
GraphQL Freshness:
  Schema last modified: 2026-04-15 (graph/)
  iOS types generated: 2026-04-12 (ios/)
  Status: STALE - iOS types are 3 days behind
  Action: Run /sync-graphql to update
```

### Check 3: Dependency Version Alignment

The district and graph submodules share some dependencies (TypeScript, Node, etc.). Major version mismatches can cause issues.

**Process:**
1. Read `district/package.json` and `graph/package.json`
2. Compare versions of shared dependencies: `typescript`, `graphql`, `drizzle-orm`, `drizzle-kit`
3. Check Node version requirements (`.nvmrc`, `package.json engines`, or `proto` config)
4. Check if both use the same package manager version (pnpm)

**Report format:**
```
Dependency Alignment:
  typescript: graph=5.8.2, district=5.8.2      OK
  drizzle-orm: graph=0.39.1, district=0.39.1   OK
  graphql: graph=16.10.0, district=16.10.0     OK
  pnpm: graph=10.6.0, district=10.6.0         OK
  node: graph=22.x, district=22.x             OK
```

### Check 4: Submodule Pointer Staleness

Check if the unified repo's submodule pointers are reasonably current compared to each submodule's remote.

**Process:**
```bash
# For each submodule, check how far behind remote
for sub in ios district graph; do
  cd "$sub"
  git fetch origin --quiet 2>/dev/null
  BEHIND=$(git rev-list --count HEAD..origin/$(git branch --show-current) 2>/dev/null || echo "?")
  LAST_UPDATE=$(git log -1 --format='%ar' 2>/dev/null)
  echo "$sub: $BEHIND commits behind remote, last updated $LAST_UPDATE"
  cd ..
done
```

**Report format:**
```
Submodule Pointers:
  ios/       0 behind remote   last updated 2 days ago    OK
  district/  12 behind remote  last updated 5 days ago    STALE
  graph/     3 behind remote   last updated 1 day ago     OK
```

Flag as STALE if more than 20 commits behind or more than 7 days old.

### Final Report

Combine all checks into a summary:

```
=== Submodule Health Report ===

DB Schema Alignment:     OK
GraphQL Freshness:       STALE (iOS 3 days behind)
Dependency Alignment:    OK
Submodule Pointers:      WARN (district/ 12 commits behind)

Recommended actions:
1. Run /sync-graphql to update iOS types
2. Run /submodule-update to pull latest district/
```

### Save Report

Determine the next sequential number in `output/cross-platform/` and save as `##-YYYY-MM-DD-health-report.md`.

## Guidelines

- **Run all checks even if one fails**: Surface all issues at once
- **Distinguish warnings from errors**: Stale pointers are warnings; schema mismatches are errors
- **Be actionable**: For every issue found, recommend a specific command or action
- **Don't alarm on intentional differences**: Some tables being in only one submodule is expected
- **Use parallel agents**: Checks 1-4 are independent and can run in parallel
